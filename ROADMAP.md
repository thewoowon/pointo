## Phase 2 · B2B SaaS 전환 (3~6개월)

> **목표**: 다른 카페/소매점도 직접 가입해서 쓸 수 있는 구조

### 핵심 기능
1. **점주 셀프 온보딩**
   - 웹 또는 앱 내 가게 등록 폼
   - `storeCode` 자동 발급
   - 무료 플랜 → 유료 플랜(Pro) 전환 흐름

2. **점주 대시보드 (웹)**
   - 회원 목록 / 스탬프 현황
   - 이벤트·쿠폰 직접 등록
   - 방문 통계 (일/주/월)

3. **콘텐츠 관리**
   - 키오스크 화면에 노출할 슬라이드 자체 등록
   - 이벤트 기간 설정

4. **멀티테넌트 격리**
   - Firestore 규칙으로 storeCode별 데이터 분리 강화
   - 이미 `storeCode` 기반 구조 존재 → 점진적 확장 가능

### 수익 모델 (가설)
| 플랜 | 가격 | 기능 |
|---|---|---|
| Free | 0원 | 회원 500명, 기본 스탬프 |
| Pro | 월 29,000원 | 무제한 회원, 통계, 커스텀 쿠폰 |
| Business | 월 79,000원 | 슬라이드 광고, 우선 지원, API |

## Phase 3 · 로컬 광고 네트워크 (6~12개월)

> **목표**: 키오스크 화면 = 지역 소상공인 광고 인벤토리

### 개념
- 가게 키오스크가 대기 화면일 때 → **인근 가게 광고/이벤트 슬라이드** 자동 노출
- 광고주(소상공인) ↔ 매체(키오스크 운영 가게) 연결 플랫폼

### 핵심 기능
1. **광고 등록 셀프서비스** (광고주)
   - 이미지/텍스트 슬라이드 업로드
   - 노출 지역·업종·기간 설정
   - 예산 충전 (CPI 또는 CPM 방식)

2. **지역 타겟팅**
   - 가게의 GPS 좌표 기반 반경 N km 내 노출
   - 업종 카테고리 매칭 (카페 키오스크 → 카페 관련 광고 우선)

3. **정산 시스템**
   - 노출 수·클릭 수 기반 광고주 과금
   - 키오스크 운영 가게에 수익 쉐어 (예: 노출 수익 30%)

4. **콘텐츠 심사**
   - 자동 심사 (부적절 이미지 필터) + 수동 승인

### 차별점
- 기존 1,000명 회원 = 이미 검증된 트래픽
- 오프라인 키오스크 기반 → 디지털 광고보다 주목도 높음
- 소상공인 대상 초저가 광고 진입장벽
